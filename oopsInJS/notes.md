### Now let's learn the oops 

## object ( and its key points )
- a real life identity with attributes(properties) and actions(methods).
- Technically => Object is a collection of properties(data) and methods (functions) that represents a real-world use_case .
- 1. Properties → Variables that store data about the object.
- 2. Methods → Functions that define the object's behavior.(ki task krna ohne)
- 3. Prototype-based → JavaScript objects can inherit properties and methods from other objects via the prototype chain.
- 4. Dynamic → You can add, modify, or delete properties/methods at runtime.
- 5. Example methods like :- toLowerCase , toUppercase , typeof etc .

## Why we use OOP
- We use because of it features . like - code organization , reusability , encapsulation , inheritance , polymorphism and easier maintence .

## Four pillars of OOP ( Main concepts/features)
- Abstraction => ( hiding the complex mechanism and just simply do the task ) . like - fetch()  method .
- Encapsulation => Keeps data safe by hiding internal details and exposing only what's neccessary through methods .
- Inheritance => Inheriting the proerties of previous class , reducing the code duplication .
- Polymorphism => Allows different objects to respond to the same method name in their own way . Example: draw() method can work differently for Circle and Square objects.
